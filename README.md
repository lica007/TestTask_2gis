## О проекте
Тестирование REST API сервиса для работы с избранными местами на карте 2GIS. Включает автоматизацию позитивных и негативных сценариев, работу с аутентификацией через cookie и валидацию JSON-схем ответов

**Тестируемый сервис:** https://regions-test.2gis.com/

![img](https://github.com/user-attachments/assets/3212da9a-f4e4-482f-8240-7192a3d1151b)

## Стек технологий
- Язык: **Java 11+**
- Фреймворк для тестов: **JUnit 5**
- Библиотека для тестирования: **REST Assured**
- Сборка и зависимости: **Gradle**
- Система контроля версий: **Git**

## Что было протестированно
В проекте тесты были структурированы по логическим группам для полного покрытия функционала API

1. **Обязательные поля:** Проверена обработки наличия, отсутствия и валидации каждого обязательного параметра (title, lat, lon)
3. **Валидация цвета:** Проверка валидации необязательного поля color
4. **Структура ответа:** Протестирован формат и целостность JSON-ответа. Проверены наличие всех заявленных полей (id, title, lat, lon, color, created_at), их типы данных и соответствие форматам
5. **Негативные сценарии:** Проверена устойчивость API к некорректным данным
6. **Граничные значения:** Проведён анализ граничных значений для полей с ограничениями
7. **Комбинированные:** Проверены взаимодействия между различными параметрами запроса

## Процедура запуска автотестов
1. Клонировать репозиторий командой: *git clone*
2. Открыть проект в **IntelliJ IDEA**
3. Запустить все тесты командой: *./gradlew clean test*
